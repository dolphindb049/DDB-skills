# Shark 2026 Q2 需求计划

## 当前已有能力

当前对外已具备的 Shark 能力，主要包括以下三类：

### `@gpu`

支持将部分用户自定义函数的计算任务编排到 GPU 执行，降低业务侧接入门槛。

当前需要进一步整理和明确可支持的算子、函数范围及约束条件，形成更清晰的对外口径。

### `gplearn`

支持因子挖掘、表达式搜索与训练加速，覆盖量化研究中的高频探索场景。

当前降频能力已经实现，但挖掘结果的业务效果还需要继续验证。现阶段可以认为算法能力已具备，业务表现仍需结合具体场景评估。

### 蒙特卡洛加速

支持蒙卡路径模拟及部分定价、风险计算加速，适用于计算量较大的金融工程场景。

相关背景可参考：[【Shark】产品单页](https://dolphindb1.atlassian.net/wiki/spaces/mfYhIEPn2ojV/pages/2547974305/Shark)

## Q2 重点方向

### 1. 内置函数 GPU 加速

推进常用内置函数的 GPU 加速能力建设，进一步扩大 GPU 可直接覆盖的函数范围。

这一方向优先级较高，目标是让用户尽量基于现有使用方式获得加速收益，减少额外改造成本，并根据实际需求逐步补充函数支持。

### 2. PDE 求解加速

补充 PDE 求解方向的 GPU 加速能力，重点包括后续计划开发的 `pdeSolver1D` 和 `pdeSolver2D` 两个函数。

该方向主要面向数值定价和相关偏微分方程求解场景，目标是提升 PDE 类计算任务的执行效率。

### 3. 回测场景 GPU 化评估

计划评估回测场景的 GPU 化空间。当前事件驱动型回测框架以状态推进和业务逻辑处理为主，向量化计算占比不高，因此不适合作为整框架 GPU 化的优先方向。

Q2 更现实的切入点是参数寻优、多组参数并行回测、批量指标计算等局部环节。相关工作以评估和识别机会点为主；如果后续希望进一步放大 GPU 收益，可能需要配合回测引擎向量化改造。

### 4. 期货、期权蒙卡定价函数加速

面向新开发的期货、期权场景，在蒙卡路径生成之后，继续推进定价函数侧的 GPU 加速。重点覆盖 payoff、路径后处理、折现与定价计算等核心环节，提升批量定价和风险计算效率。

该方向放在当前优先级的后段，待前述通用能力和 PDE 方向推进后再进一步展开。

## 当前边界

- 华为 GPU 生态目前暂不支持，不作为 Q2 重点投入方向。

## Q2 预期产出

- 落地一批具备通用价值的内置函数 GPU 加速能力。
- 完成 `pdeSolver1D`、`pdeSolver2D` 相关加速方案设计或原型验证。
- 形成回测场景 GPU 化的评估结论，并明确参数寻优场景下可优先落地的 GPU 加速方案。
- 完成面向期货、期权蒙卡路径后的定价函数加速方案与原型验证。
