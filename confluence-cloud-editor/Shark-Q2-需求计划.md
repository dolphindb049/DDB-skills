# Shark Q2 需求计划

## 当前已有能力

当前对外已具备的 Shark 能力，主要包括以下三类：

### `@gpu`

支持将适合并行的计算任务编排到 GPU 执行，降低业务侧接入门槛。

### `gplearn`

支持因子挖掘、表达式搜索与训练加速，覆盖量化研究中的高频探索场景。

### 蒙特卡洛加速

支持蒙卡路径模拟及部分定价、风险计算加速，适用于计算量较大的金融工程场景。

相关背景可参考：[【Shark】产品单页](https://dolphindb1.atlassian.net/wiki/spaces/mfYhIEPn2ojV/pages/2547974305/Shark)

## Q2 重点方向

### 1. 回测框架 GPU 化评估

计划评估回测框架的 GPU 化空间。整体回测流程更偏事件驱动、状态推进和业务逻辑处理，属于逻辑密集型而非计算密集型场景，未必适合作为整框架 GPU 化的优先方向。

但在参数寻优、多组参数并行回测、批量指标计算等环节，存在较明确的加速空间。Q2 目标是完成可行性评估，识别哪些局部计算模块适合卸载到 GPU，并优先聚焦参数寻优相关场景。

### 2. 内置函数 GPU 加速

推进常用内置函数的 GPU 加速能力建设，进一步扩大 GPU 可直接覆盖的函数范围。目标是让用户尽量基于现有使用方式获得加速收益，减少额外改造成本。

### 3. PDE 求解加速

补充 PDE 求解方向的 GPU 加速能力，重点包括后续计划开发的 `pdeSolver1D` 和 `pdeSolver2D` 两个函数。该方向主要面向数值定价和相关偏微分方程求解场景，提升 PDE 类计算任务的执行效率。

### 4. 期货、期权蒙卡定价函数加速

面向新开发的期货、期权场景，在蒙卡路径生成之后，继续推进定价函数侧的 GPU 加速。重点覆盖 payoff、路径后处理、折现与定价计算等核心环节，提升批量定价和风险计算效率。

## Q2 预期产出

- 形成回测框架 GPU 化的评估结论与优先级建议。
- 明确参数寻优场景下可优先落地的 GPU 加速方案。
- 落地一批具备通用价值的内置函数 GPU 加速能力。
- 完成 `pdeSolver1D`、`pdeSolver2D` 相关加速方案设计或原型验证。
- 完成面向期货、期权蒙卡路径后的定价函数加速方案与原型验证。
